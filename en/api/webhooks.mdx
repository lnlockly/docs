---
title: "Webhooks"
description: "Outbound event subscriptions and internal service-to-service endpoints."
---

AgentFlow emits outbound webhooks for events on your account (call settled, payout confirmed, subscription grant issued) and exposes a small set of internal endpoints for sibling services. There are no inbound payment-provider webhooks — payment flows are settled on-chain via the subscription contract and the chain indexer.

## Outbound events

Subscribe a URL via the Cabinet (**Settings → Webhooks**) or via API:

```bash
curl -X POST https://api.agentflow.website/me/webhooks \
  -H "Content-Type: application/json" \
  -H "Cookie: af_session=..." \
  -d '{
    "url": "https://your-app.example/agentflow",
    "events": ["call.settled", "payout.confirmed", "subscription.granted"]
  }'
```

| Event | Payload highlight |
|---|---|
| `call.settled` | `agentSlug`, `flowDebited`, `creatorShare`, `holderShare` |
| `payout.confirmed` | `payoutId`, `txHash`, `netFlow` |
| `subscription.granted` | `tier`, `monthlyGrantFlow`, `renewsAt` |
| `token.graduated` | `tokenSlug`, `dexPairAddress` |

## Signing and verification

Each delivery includes an `X-AgentFlow-Signature` header — an HMAC-SHA256 of the raw body using your subscription secret. Reject any request whose signature does not match.

```typescript
import { createHmac, timingSafeEqual } from "node:crypto";

function verify(rawBody: string, signature: string, secret: string) {
  const expected = createHmac("sha256", secret).update(rawBody).digest("hex");
  const a = Buffer.from(expected, "hex");
  const b = Buffer.from(signature, "hex");
  return a.length === b.length && timingSafeEqual(a, b);
}
```

## Retries

Failed deliveries (non-2xx response, network error) are retried with exponential backoff: 30 s, 5 min, 30 min, 2 h, 6 h. After five failures the subscription enters `paused` state — the Cabinet surfaces a banner and you can replay events.

## Idempotency

Each delivery carries an `X-AgentFlow-Delivery` UUID. Treat handlers as idempotent on this ID — duplicate deliveries are accepted with `200 OK` but should produce no second side effect on your end.

## Internal endpoints

The API exposes `/internal/*` routes consumed by sibling services (agentflow-bot, agentflow-runtime, agentflow-agents). Auth is via the `X-Internal-Token` header and a shared secret. These routes are **not** for public integrators.

| Route | Used by |
|---|---|
| `POST /internal/users/upsert` | agentflow-bot |
| `POST /internal/aiturns` | agentflow-runtime |
| `POST /internal/marketplace-notifications` | agentflow-agents |
| `POST /internal/leads` | agentflow-bot |

<Warning>
  Never expose internal endpoints to the public internet. They bypass user-level authorization and must run inside a private network or service mesh.
</Warning>
