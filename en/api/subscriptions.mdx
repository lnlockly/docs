## POST /subscriptions/checkout

Initiate a subscription checkout flow. **Auth**: required.

<ParamField body="tier" type="string" required>
  One of `plus`, `pro`, `max`.
</ParamField>

```bash
curl -X POST https://api.agentflow.website/subscriptions/checkout \
  -H "Content-Type: application/json" \
  -H "Cookie: af_session=..." \
  -d '{ "tier": "pro" }'
```

**Response**

```json
{
  "subscriptionId": "sub_01HQ...",
  "tier": "pro",
  "contractAddress": "0x...",
  "amountFlow": 100,
  "payload": "0x...",
  "expiresAt": "2026-04-25T11:30:00Z"
}
```

Pass `contractAddress`, `amountFlow`, and `payload` to the user's wallet to build the on-chain transaction. Activation completes once the chain indexer detects the deposit. Pre-launch, `contractAddress` is `null` and the platform debits the off-chain ledger directly.

## GET /me/subscription

Fetch the current user's active subscription. **Auth**: required.

```bash
curl https://api.agentflow.website/me/subscription \
  -H "Cookie: af_session=..."
```

**Response**

```json
{
  "tier": "pro",
  "status": "active",
  "monthlyGrantFlow": 100,
  "currentGrantRemainingFlow": 73.4,
  "renewsAt": "2026-05-25T11:30:00Z",
  "txHash": "0x..."
}
```

`status` values: `active`, `grace` (failed renewal, retrying), `canceled`, `none`.

<ParamField body="rail" type="string" required>
  `wallet` or `manual`.
</ParamField>

```bash
curl -X POST https://api.agentflow.website/me/payouts/request \
  -H "Content-Type: application/json" \
  -H "Cookie: af_session=..." \
  -d '{ "amountFlow": 100, "rail": "wallet" }'
```

The destination address is taken from your SIWE-verified wallet.

## GET /me/payouts

List the user's payouts.

```bash
curl https://api.agentflow.website/me/payouts \
  -H "Cookie: af_session=..."
```

**Response**: array of payout objects with `id`, `amountFlow`, `feeFlow`, `rail`, `status`, `createdAt`.

## GET /me/payouts/:id

Single payout detail, including rail-specific tx hash if `sent` or `confirmed`.

## POST /me/payouts/:id/cancel

Cancel a `pending` payout and re-credit the $FLOW.

## Errors

| Code | When |
|---|---|
| `tier_unknown` | Bad tier value |
| `subscription_active` | New checkout while one is already active |
| `withdraw_too_low` | Below the rail's minimum |
| `withdraw_too_high` | Above withdrawable balance |
| `wallet_not_connected` | Wallet rail without a SIWE-linked address |
| `payout_immutable` | Cancel attempted on `sent`/`confirmed` payout |
