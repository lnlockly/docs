`FlowGrowToken` — тикер **GWT** — это второй токен шаблона `dpnm`. Существует, чтобы компенсировать пользователям комиссии, которые они платят. Каждый доллар уплаченной комиссии (buy fee, sell fee, стоимость продления, активации) минтит `1 GWT` плательщику.

GWT **не** торгуется в закрытой системе, **не** имеет пары с USDT и **не** инвестиционный инструмент. Это redemption-кредит: GWT можно обменять на лимит дохода внутри `FlowProtocol`.

Контракт GWT в BSC testnet: [`0x4071...0783`](https://testnet.bscscan.com/address/0x40711d364b680ad7e85429bd36d7237E4e960783).

## Заработок GWT

Каждый вызов с комиссией минтит GWT caller — 1:1 в долларах:

| Действие | Уплаченная комиссия | GWT минтится |
|---|---|---|
| `buy(100 USDT)` | `$20` (`20%` от покупки) | `20 GWT` |
| `sell` на `$100` | `$10` (`10%` от ценности) | `10 GWT` |
| `activate(...)` | `$10` стоимость активации | `10 GWT` |
| `extendTree(1)` | `$10` стоимость продления | `10 GWT` |
| `extendTree(3)` | `$30` стоимость продления | `30 GWT` |

GWT не нужно вручную claim для buy и sell — он минтится автоматически. Для накопленных tree-маркетинговых выплат, в которых сидит ваш GWT, вызовите `claimGWT()`:

```typescript
await flowProtocol.claimGWT();
// весь pending GWT с комиссий ваших активных потомков попадает в кошелёк
```

## Обмен GWT на лимит дохода

Лимит дохода ограничивает заработок `2x` от того, сколько вы заплатили за `$FLOW`. Если потолок достигнут и вы хотите продолжать извлекать ценность, можно обменять GWT:

```typescript
const gwtToRedeem = parseUnits("100", 18);
// Approve GWT для FlowProtocol
await gwt.approve(flowProtocol.address, gwtToRedeem);
// Заплатить $2 USDT redemption fee
await usdt.approve(flowProtocol.address, parseUnits("2", 18));
// Redeem
await flowProtocol.buyIncomeLimitWithGWT(gwtToRedeem);
```

| Параметр | Значение |
|---|---|
| Курс обмена | `1 GWT → $1.25` лимита дохода |
| Redemption fee | `$2 USDT` за вызов |
| Cap на вызов | `10%` от пожизненно полученного дохода caller |

Итого `100 GWT` на redemption → `$125` лимита дохода добавляется к аккаунту, плюс `$2` USDT.

Cap 10% означает, что пользователь, заработавший пожизненно `$10 000`, может redeem максимум `$1 000` лимита за один вызов — никаких флэш-обменов огромных мешков GWT в одной транзакции.

## Откуда берётся GWT

GWT минтится протоколом в момент уплаты комиссии. Фиксированной supply нет — она расширяется по мере того, как протокол собирает комиссии. Идея: каждый доллар, который вы отдаёте системе, возвращается вам форвард-кредитом, способным сдвинуть ваш будущий потолок дохода.

В здоровой сети с непрерывной активностью обычный пользователь накапливает больше GWT, чем нужно для redemption. Излишек просто остаётся в кошельке — пригодится для будущих пополнений лимита либо просто как запись о вкладе пользователя в комиссионный поток протокола.

## Почему не возвращать комиссии в USDT напрямую

Возвращение USDT свело бы смысл комиссий к нулю — они существуют, чтобы (a) кормить феноменальное дерево, (b) удерживать стоимость в пуле, (c) финансировать treasury. GWT — это *будущая* компенсация: она поднимает потолок заработка, но только когда вы её активно тратите. Большую часть времени GWT просто копится как латентный headroom.
