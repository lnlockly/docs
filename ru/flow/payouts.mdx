Выплаты — то, как заработанный FLOW превращается обратно в реальные деньги. Поддерживается три рельса.

## Рельсы

| Рельс | Минимум | Комиссия | Срок |
|---|---|---|---|
| USDT BEP20 | 10 FLOW | 1 FLOW + комиссия сети | ~5 мин |
| CryptoBot | 5 FLOW | 1% | Мгновенно |
| Ручной (юрлицо) | 1000 FLOW | Бесплатно | 3–5 рабочих дней |

USDT BEP20-выплаты идут на адрес кошелька, который пользователь добавляет в Cabinet. CryptoBot-выплаты идут в привязанный Telegram. Ручные выплаты требуют пройти KYB и предоставить инвойс-данные.

## Запрос выплаты

```bash
curl -X POST https://api.agentflow.website/me/payouts/request \
  -H "Content-Type: application/json" \
  -H "Cookie: af_session=..." \
  -d '{
    "amountFlow": 100,
    "rail": "usdt_bep20",
    "address": "0x..."
  }'
```

<ParamField path="amountFlow" type="number" required>
  Сумма к выводу в FLOW. Должна быть ≤ `withdrawableFlow` из эндпоинта баланса.
</ParamField>

<ParamField path="rail" type="string" required>
  Одно из `usdt_bep20`, `cryptobot`, `manual`.
</ParamField>

<ParamField path="address" type="string">
  Обязательно для `usdt_bep20`. Кошелёк-получатель.
</ParamField>

## Ответ

```json
{
  "payoutId": "pay_01HQ...",
  "amountFlow": 100,
  "feeFlow": 1,
  "netFlow": 99,
  "rail": "usdt_bep20",
  "status": "pending",
  "createdAt": "2026-04-25T12:00:00Z"
}
```

FLOW списывается с баланса сразу. Выплата встаёт в `pending` и переходит в `sent`, когда рельс подтверждает. Если рельс отклонил (некорректный адрес, нехватка ликвидности), выплата откатывается и FLOW возвращается на баланс.

## Статусы

| Статус | Значение |
|---|---|
| `pending` | Отправлено, ждёт обработки рельсом |
| `sent` | Деньги ушли с платформы, ждём подтверждения |
| `confirmed` | Рельс подтвердил доставку |
| `failed` | Рельс отклонил; FLOW возвращён |
| `canceled` | Пользователь отменил до отправки |

## Список выплат

```bash
curl https://api.agentflow.website/me/payouts \
  -H "Cookie: af_session=..."
```

Возвращает историю выплат пользователя со статусами и деталями рельсов.

## Отмена pending

```bash
curl -X POST https://api.agentflow.website/me/payouts/pay_01HQ.../cancel \
  -H "Cookie: af_session=..."
```

Отменить можно только `pending`-выплаты. После `sent` отмена невозможна.

<Warning>
  Выплаты **необратимы** после перехода в `sent`. Дважды проверяйте кошелёк-получатель перед отправкой. Платформа не может вернуть средства, отправленные на некорректный или вражеский адрес.
</Warning>

## Compliance

Выплаты сверх установленных платформой порогов могут включить проверку источника средств. Заработанный FLOW (доля создателя, награды держателям, фриланс) проходит эти проверки быстрее, чем top-up FLOW, потому что у платформы есть полная история происхождения.
