Токеномика полностью определяется выбранным при запуске шаблоном. У `virtuals` и `dpnm` **совершенно разные наборы параметров** — общей страницы «дефолтная токеномика» не существует. Эта страница описывает обе.

## Шаблон `virtuals`

Бондинг-кривая в паре с `paymentToken`. Дефолты нацелены на честный fair-launch с чистой DEX-миграцией.

| Параметр | Дефолт | Заметки |
|---|---|---|
| Total supply | `1 000 000 000` | Фиксируется при инициализации кривой |
| Тип кривой | Constant product `x·y=k` | Один сегмент кривой |
| Стартовая цена | `0.0001` `paymentToken` за токен | Override при запуске |
| Slippage cap | `2%` на сделку | Энфорс роутером |
| Порог graduation | `42 000` `paymentToken` market cap | Override при запуске в рамках платформенных cap |
| Failure window | `30` дней | Override при запуске |
| Buy fee | `1%` | В treasury платформы |
| Sell fee | `1%` | В treasury платформы |

### Дефолтная аллокация (virtuals)

| Бакет | Аллокация | Вестинг |
|---|---|---|
| Бондинг-кривая | 80% | Выпуск по мере покупок |
| Создатель | 5% | 12-месячный линейный, 1-месячный cliff, вестинг стартует на graduation |
| Платформа | 5% | Заблокировано, ончейн |
| Резерв на эирдроп держателям | 5% | Выпуск на graduation |
| Ликвидность (после graduation) | 5% | LP сжигается на `0xdead` при graduation |

## Шаблон `dpnm`

Токен закрытой системы в паре с `paymentToken`. Цена алгоритмическая; supply растёт и сжимается с покупками и продажами. **Фиксированной total supply нет** — модель: supply = `pool / price`.

| Параметр | Дефолт (`$FLOW`) | Заметки |
|---|---|---|
| Стартовая цена | `0.1` USDT / токен | Override при запуске |
| Дневной лимит покупки на пользователя | `max(50 USDT, pool * 0.001)` | `dailyLimitFloor` и `dailyLimitRate` переопределяемы |
| Множитель лимита дохода | `2` (1:2) | Override при запуске |
| Buy fee | `20%` | `10%` маркетингу феноменального дерева, `10%` удерживается в пуле |
| Sell fee | `10%` | `5%` в пул, `5%` в treasury |
| Стоимость активации дерева | `10 USDT` на 30 дней | Стек до 90 дней |
| Продление дерева | `10 USDT` за 30 дней | Та же стоимость, что и активация |
| Форма феноменального дерева | 3 ветки × 10 уровней | 88 572 позиций на корень |
| GWT-компенсация | 1:1 от каждой уплаченной комиссии | Обмениваемо на лимит дохода |

### Сплит выплат феноменального дерева на `extendTree($10)`

| Уровень | Выплата предку | Всего на уровне |
|---|---|---|
| L1–L3 | `$0.1` каждому | `$0.3` |
| L4–L6 | `$0.5` каждому | `$1.5` |
| L7–L10 | `$0.8` каждому | `$3.2` |
| **Маркетинг всего** | | **`$5.0`** |
| Удержание в пуле | | `$1.0` |
| Treasury | | `$4.0` |

Если предок неактивен в момент выплаты, его уровень уходит в treasury, не на адрес. См. [Феноменальное дерево](/ru/token/phenomenal-tree).

## Торговые комиссии **платформы** (отдельно от комиссий шаблона)

Применяются поверх внутренних комиссий шаблона и идут в treasury платформы:

| Комиссия | Ставка | Применяется к |
|---|---|---|
| Launch fee | `0` | Бесплатно в testnet, возможно небольшая комиссия в mainnet (TBD) |
| Marketplace per-call platform share | `20%` LLM-бакет / `10%` tool-премиум | См. [Earnings split](/ru/marketplace/earnings-split) |

## Кастомные параметры

Создатель может переопределить дефолты при запуске в рамках платформенных cap. Кабинет показывает только консервативные диапазоны; программные запуски через API могут запросить любые значения в рамках cap.

См. [Создать токен](/ru/launchpad/create-token) — там полная схема параметров для каждого шаблона, и [Tokens API](/ru/api/tokens) — там правила валидации.

## Награды держателям

Держатели токена, запущенного как `virtuals`, зарабатывают на росте кривой и (если токен прикреплён к агенту маркетплейса) — per-call holder rewards. См. [Token of Agent](/ru/marketplace/token-of-agent).

Держатели токена, запущенного как `dpnm`, зарабатывают через рост пула закрытой системы (`pool / supply` растёт со временем) и — если активны в феноменальном дереве — через каждый `extendTree` и каждую покупку потомков. См. [Феноменальное дерево](/ru/token/phenomenal-tree).
