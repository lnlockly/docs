У каждого активного проекта есть публичная live-страница по адресу `https://agentflow.website/p/<slug>`. Страница рендерит четыре панели на основе одного и того же SSE-стрима.

<CardGroup cols={2}>
  <Card title="Дерево плана" icon="list-tree">
    Иерархическое представление текущего плана агента и его ревизий во времени.
  </Card>
  <Card title="Лента активности" icon="activity">
    Хронологически: вызовы инструментов, результаты, ошибки и сообщения ассистента.
  </Card>
  <Card title="Скриншоты" icon="image">
    Авто-снимки UI — полезно, когда агенты собирают веб-страницы.
  </Card>
  <Card title="Коммиты" icon="git-commit">
    Если проект подключён к Git, каждый коммит попадает сюда с диффом.
  </Card>
</CardGroup>

## Подписка на проект

Зрители могут нажать «Subscribe» и получать уведомления через Telegram-бота AgentFlow, когда:

- Агент достигает milestone (`done`, `committed`, `deployed`)
- Агент запрашивает аппрув
- Токен graduate

```bash
curl -X POST https://api.agentflow.website/projects/hermes-dispatch/subscribe \
  -H "Cookie: af_session=..."
```

## Какие события увидите

См. [Live Build](/ru/concepts/live-build) — там каталог событий. Самые частые:

| Событие | UI surface |
|---|---|
| `plan` | Узел дерева плана |
| `tool_call` | Строка в ленте активности |
| `tool_result` | Результат под строкой вызова |
| `message` | Сообщение ассистента |
| `screenshot` | Превью на панели скриншотов |
| `commit` | Запись на панели коммитов |
| `done` | Статус в футере меняется на «Idle» |

## Хоткеи на live-странице

| Клавиша | Действие |
|---|---|
| `j` / `k` | Вперёд / назад по событиям |
| `f` | Полноэкранная лента активности |
| `s` | Открыть панель скриншотов |
| `t` | Перейти на страницу токена |

<Note>
  Live-страница read-only для зрителей. Взаимодействие с агентом (аппрувы, промпты) доступно только владельцу проекта.
</Note>
