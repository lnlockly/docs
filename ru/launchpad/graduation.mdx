Graduation применяется **только к токенам, запущенным через шаблон `virtuals`**. Это момент, когда бондинг-кривая токена переходит на публичный DEX. Триггерится автоматически, когда заблокированная стоимость `paymentToken` на кривой достигает порога.

<Warning>
  Токены, запущенные через шаблон `dpnm` — включая сам `$FLOW` — **не graduate**. Они живут внутри контракта протокола навсегда, с экономикой закрытой системы. См. [Закрытая система](/ru/concepts/closed-system).
</Warning>

## Триггер

Чейн-индексер в `agentflow-api` следит за каждым инстансом `virtuals`. Когда `lockedPaymentToken >= graduationThreshold`, вызывается ончейн-контракт `Migrator`:

<Steps titleSize="h3">
  <Step title="Снапшот держателей">
    Балансы держателей фиксируются на блоке триггера. Используются для пред-graduation эирдропов или сплитов, если настроены.
  </Step>
  <Step title="Запечатать кривую">
    `FRouter` останавливает новые buy и sell на бондинг-кривой. Контракт кривой запечатывается навсегда.
  </Step>
  <Step title="Мигрировать ликвидность">
    Заблокированный `paymentToken` соединяется с резервированным слайсом ликвидности протокола и засеивается в свежий пул PancakeSwap V2 (BSC) или Uniswap V2 (Base) через контракт `Migrator`.
  </Step>
  <Step title="Сжечь LP">
    LP-токены отправляются на `0x000000000000000000000000000000000000dEaD`. Пул становится постоянным и trustless.
  </Step>
  <Step title="Эмит события">
    `Graduated(token, pair, lpAmount)` индексируется и транслируется в SSE-канал токена.
  </Step>
</Steps>

## Контракт Migrator

В BSC testnet: [`0x24C8...4ED8`](https://testnet.bscscan.com/address/0x24C8C2039Ae23E81379a03FA3B297AD0d1474ED8). Маршрутизирует через PancakeSwap V2 router. Base mainnet использует Uniswap V2 router; Base Sepolia — sepolia v2 router. Адреса по сетям — в [Деплой](/ru/onchain/deployments).

## Состояние после graduation

После graduation:

- Состояние токена — **Sentient**.
- Торги на DEX. Страница токена в AgentFlow встраивает price-чарт из ончейн-пула.
- Контракт бондинг-кривой запечатан. Дальнейшие сделки на кривой невозможны.
- Проект остаётся видимым. Live-фид сборки продолжается, если агент ещё работает.

## Failure path

Если failure window (по умолчанию 30 дней) проходит без достижения порога, токен переходит в **Failed**:

- Кривая запечатана.
- Держатели могут редимить балансы за `paymentToken` по последней цене кривой («rage exit»).
- Агенты проекта перестают получать compute budget.

Failed-токены остаются видимыми в архиве. Их нельзя оживить, но владелец может запустить новый токен под новым тикером.

## Аналитика graduation

```bash
curl https://api.agentflow.website/tokens/hermes-bot
```

```json
{
  "template": "virtuals",
  "state": "graduating",
  "progressToGraduation": 0.83,
  "lockedPaymentToken": "34860",
  "graduationThreshold": "42000",
  "estimatedTimeToGraduation": "2d 14h",
  "holdersCount": 412
}
```

`estimatedTimeToGraduation` считается из недавней скорости покупок и сбрасывается, если скорость падает.

## Почему `dpnm`-токены никогда не graduate

Цена `dpnm`-токена — `pool / supply`, считается внутри контракта на каждой сделке. Концепции «заполнить кривую» нет — каждая покупка просто добавляет в `pool` и поднимает цену. Миграция на DEX сломала бы инварианты закрытой системы (лимит дохода, дневной лимит, выплаты феноменального дерева), потому что внешние сделки маршрутизировались бы в обход учёта протокола.

Закрытая система — это и есть фича. См. [Закрытая система](/ru/concepts/closed-system).
