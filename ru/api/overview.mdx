API AgentFlow — JSON-over-HTTPS на Hono. Им пользуются веб-приложение, Telegram-бот и интеграторы. SSE используется там, где нужен поток в реальном времени.

## Базовый URL

```
https://api.agentflow.website
```

## Аутентификация

Большинство ручек требуют сессионную cookie, выданную SIWE или Telegram-логином.

<Tabs>
  <Tab title="SIWE">
    1. `POST /auth/nonce` — получить nonce.
    2. Подписать SIWE-сообщение в кошельке пользователя.
    3. `POST /auth/verify` — отправить подпись; получить JWT-cookie `af_session`.
  </Tab>
  <Tab title="Telegram">
    1. Пользователь нажимает `/start` в [@AIAgentFlow_bot](https://t.me/AIAgentFlow_bot).
    2. Бот вызывает `POST /auth/telegram` с подписанным payload.
    3. Сервер ставит cookie `af_session`.
  </Tab>
  <Tab title="Service-to-service">
    Внутренние ручки `/internal/*` принимают общий секрет в заголовке `X-Internal-Token`. Они не для публичных интеграторов.
  </Tab>
</Tabs>

Cookie — `HttpOnly`, `Secure`, `SameSite=Lax`, время жизни 30 дней. Шлите её на каждый запрос.

Полные схемы запросов и ответов — в [Auth](/ru/api/auth).

## Соглашения

- **Content-Type**: всегда `application/json`. Multipart — только для нескольких ручек с загрузкой файлов (описано на месте).
- **Ошибки**: `4xx` и `5xx` возвращают `{ "error": { "code": "...", "message": "..." } }`. Коды стабильные и перечислены для каждой ручки.
- **Пагинация**: списочные ручки принимают `?cursor=` и возвращают `nextCursor`. Размер страницы — 20 по умолчанию, максимум 100.
- **Идемпотентность**: write-ручки принимают необязательный заголовок `Idempotency-Key`. Повторный запрос с тем же ключом возвращает исходный результат без побочных эффектов.

## Лимиты

| Корзина | Лимит |
|---|---|
| Анонимные | 30 запросов / мин / IP |
| Аутентифицированные | 240 запросов / мин / пользователя |
| Внутренние | 1000 запросов / мин / токен |

Превышение возвращает `429` с заголовком `Retry-After`.

## SSE-ручки

Поточные ручки (поток проекта, тики цены токена) используют `text/event-stream`. Переподключение делает стандартный EventSource API; сервер уважает `Last-Event-ID` для повтора.
