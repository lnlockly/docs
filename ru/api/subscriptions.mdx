Подписки в AgentFlow теперь — **feature gates**, а не выдача кредитов. Прежняя модель ежемесячного гранта FLOW-кредитов исчезла вместе с переездом `$FLOW` в ончейн-токен. Подписки открывают платформенные фичи для пользователей маркетплейса и live-сборки.

<Note>
  Старые доки ссылались на `monthlyGrantFlow` и `currentGrantRemainingFlow`. Этих полей больше нет. Если у вас биллинг зависел от них, переключайтесь на проверки фич (`features.priorityQueue`, `features.premiumAgents` и т. п.).
</Note>

## Тарифы

| Тариф | Стоимость в месяц | Фичи |
|---|---|---|
| **Plus** | $20 | Приоритетная очередь, базовая аналитика |
| **Pro** | $100 | Всё из Plus + premium-агенты, продвинутая аналитика, ускоренный SSE |
| **Max** | $200 | Всё из Pro + приватные страницы проектов, выделенный runtime-слот |

Смена тарифа на даунгрейд срабатывает в начале следующего расчётного периода, на апгрейд — сразу (pro-rated).

## POST /subscriptions/checkout

Стартует флоу checkout подписки. **Auth**: нужен.

<ParamField body="tier" type="string" required>
  Один из `plus`, `pro`, `max`.
</ParamField>

<ParamField body="provider" type="string" required>
  Один из `cryptobot`, `platega`, `usdt_bep20`.
</ParamField>

```bash
curl -X POST https://api.agentflow.website/subscriptions/checkout \
  -H "Content-Type: application/json" \
  -H "Cookie: af_session=..." \
  -d '{ "tier": "pro", "provider": "cryptobot" }'
```

**Ответ**

```json
{
  "subscriptionId": "sub_01HQ...",
  "tier": "pro",
  "provider": "cryptobot",
  "checkoutUrl": "https://t.me/CryptoBot?start=invoice-...",
  "expiresAt": "2026-04-26T11:30:00Z"
}
```

`checkoutUrl` зависит от провайдера. Редирекьте пользователя туда. Активация происходит на сервере, как только webhook провайдера подтверждает оплату.

## GET /me/subscription

Получить активную подписку текущего пользователя. **Auth**: нужен.

```bash
curl https://api.agentflow.website/me/subscription \
  -H "Cookie: af_session=..."
```

**Ответ**

```json
{
  "tier": "pro",
  "status": "active",
  "renewsAt": "2026-05-26T11:30:00Z",
  "provider": "cryptobot",
  "features": {
    "priorityQueue": true,
    "premiumAgents": true,
    "fasterSse": true,
    "privateProjects": false,
    "dedicatedRuntime": false
  }
}
```

Значения `status`: `active`, `grace` (renew не прошёл, ретраим), `canceled`, `none`.

## POST /me/subscription/cancel

Отменить в конце периода. Пользователь остаётся на тарифе до `renewsAt`, потом падает в `none`.

```bash
curl -X POST https://api.agentflow.website/me/subscription/cancel \
  -H "Cookie: af_session=..."
```

## Авто-renew и grace

Подписки авто-обновляются через того же провайдера, через которого активировались. Если оплата renewal проваливается, подписка переходит в 7-дневный grace, в течение которого фичи живые, а оплата ретраится ежедневно. Через 7 дней неудачных ретраев подписка отменяется.

## Errors

| Code | When |
|---|---|
| `tier_unknown` | Плохое значение tier |
| `provider_unsupported` | Провайдер не включён в этом деплое |
| `subscription_active` | Новый checkout, когда уже есть активный |

<Note>
  Withdrawals больше не обрабатываются этой группой эндпоинтов. С `$FLOW` ончейн «withdrawal» — это просто продажа токенов через `FlowProtocol.sell(...)` прямо с кошелька в USDT-пару. См. [Buy / Sell](/ru/token/buy-sell). Старые эндпоинты `/me/payouts` депрекейтнуты.
</Note>
