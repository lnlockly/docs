## POST /subscriptions/checkout

Запустить активацию подписки. **Auth**: требуется.

<ParamField body="tier" type="string" required>
  Одно из `plus`, `pro`, `max`.
</ParamField>

```bash
curl -X POST https://api.agentflow.website/subscriptions/checkout \
  -H "Content-Type: application/json" \
  -H "Cookie: af_session=..." \
  -d '{ "tier": "pro" }'
```

**Ответ**

```json
{
  "subscriptionId": "sub_01HQ...",
  "tier": "pro",
  "contractAddress": "0x...",
  "amountFlow": 100,
  "payload": "0x...",
  "expiresAt": "2026-04-25T11:30:00Z"
}
```

Передайте `contractAddress`, `amountFlow` и `payload` в кошелёк — он соберёт on-chain транзакцию. Активация завершится, когда индексер увидит депозит. До запуска основной сети `contractAddress` равен `null`, и платформа списывает $FLOW из регистра напрямую.

## GET /me/subscription

Получить активную подписку текущего пользователя. **Auth**: требуется.

```bash
curl https://api.agentflow.website/me/subscription \
  -H "Cookie: af_session=..."
```

**Ответ**

```json
{
  "tier": "pro",
  "status": "active",
  "monthlyGrantFlow": 100,
  "currentGrantRemainingFlow": 73.4,
  "renewsAt": "2026-05-25T11:30:00Z",
  "txHash": "0x..."
}
```

Значения `status`: `active`, `grace` (продление не прошло, идут повторы), `canceled`, `none`.

## POST /me/payouts/request

Вывести $FLOW. Полный поток — в [Выплатах](/ru/flow/payouts).

<ParamField body="amountFlow" type="number" required>
  Сумма в $FLOW. Не больше доступного к выводу баланса.
</ParamField>

<ParamField body="rail" type="string" required>
  `wallet` или `manual`.
</ParamField>

```bash
curl -X POST https://api.agentflow.website/me/payouts/request \
  -H "Content-Type: application/json" \
  -H "Cookie: af_session=..." \
  -d '{ "amountFlow": 100, "rail": "wallet" }'
```

Адрес-получатель — ваш SIWE-кошелёк.

## GET /me/payouts

Список выплат пользователя.

```bash
curl https://api.agentflow.website/me/payouts \
  -H "Cookie: af_session=..."
```

**Ответ**: массив объектов выплат с `id`, `amountFlow`, `feeFlow`, `rail`, `status`, `createdAt`.

## GET /me/payouts/:id

Детали одной выплаты, включая tx-хеш для `sent` или `confirmed`.

## POST /me/payouts/:id/cancel

Отменить `pending`-выплату и вернуть $FLOW на баланс.

## Ошибки

| Код | Когда |
|---|---|
| `tier_unknown` | Неверное значение тарифа |
| `subscription_active` | Запрос на новую активацию, когда уже активна |
| `withdraw_too_low` | Меньше минимума рельса |
| `withdraw_too_high` | Больше доступного к выводу баланса |
| `wallet_not_connected` | Запрос на рельс `wallet` без подключённого SIWE-адреса |
| `payout_immutable` | Попытка отмены `sent`/`confirmed` |
