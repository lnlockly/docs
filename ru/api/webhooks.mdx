---
title: "Вебхуки"
description: "Исходящие события подписок и внутренние ручки между сервисами."
---

AgentFlow отправляет исходящие вебхуки на события вашего аккаунта (вызов рассчитан, выплата подтверждена, начислен грант подписки) и держит небольшой набор внутренних ручек для соседних сервисов. Входящих вебхуков от платёжных провайдеров нет — оплата идёт on-chain через контракт подписки и индексер блокчейна.

## Исходящие события

Подпишите URL через Cabinet (**Settings → Webhooks**) или через API:

```bash
curl -X POST https://api.agentflow.website/me/webhooks \
  -H "Content-Type: application/json" \
  -H "Cookie: af_session=..." \
  -d '{
    "url": "https://your-app.example/agentflow",
    "events": ["call.settled", "payout.confirmed", "subscription.granted"]
  }'
```

| Событие | Что в payload |
|---|---|
| `call.settled` | `agentSlug`, `flowDebited`, `creatorShare`, `holderShare` |
| `payout.confirmed` | `payoutId`, `txHash`, `netFlow` |
| `subscription.granted` | `tier`, `monthlyGrantFlow`, `renewsAt` |
| `token.graduated` | `tokenSlug`, `dexPairAddress` |

## Подпись и проверка

К каждой доставке прикладывается заголовок `X-AgentFlow-Signature` — HMAC-SHA256 от тела запроса с вашим секретом подписки. Отклоняйте запросы с неверной подписью.

```typescript
import { createHmac, timingSafeEqual } from "node:crypto";

function verify(rawBody: string, signature: string, secret: string) {
  const expected = createHmac("sha256", secret).update(rawBody).digest("hex");
  const a = Buffer.from(expected, "hex");
  const b = Buffer.from(signature, "hex");
  return a.length === b.length && timingSafeEqual(a, b);
}
```

## Повторы

Неудачные доставки (ответ не 2xx, ошибка сети) повторяются с экспоненциальной задержкой: 30 с, 5 мин, 30 мин, 2 ч, 6 ч. После пяти неудач подписка ставится в `paused` — Cabinet показывает баннер, события можно проиграть заново.

## Идемпотентность

Каждая доставка несёт UUID `X-AgentFlow-Delivery`. Делайте обработчик идемпотентным по этому ID — дубли принимаются с `200 OK`, но не должны вызывать второй побочный эффект на вашей стороне.

## Внутренние ручки

API экспонирует маршруты `/internal/*` для соседних сервисов (agentflow-bot, agentflow-runtime, agentflow-agents). Авторизация — через заголовок `X-Internal-Token` и общий секрет. Эти маршруты **не** для публичных интеграторов.

| Маршрут | Используется |
|---|---|
| `POST /internal/users/upsert` | agentflow-bot |
| `POST /internal/aiturns` | agentflow-runtime |
| `POST /internal/marketplace-notifications` | agentflow-agents |
| `POST /internal/leads` | agentflow-bot |

<Warning>
  Не выставляйте внутренние ручки в открытый интернет. Они обходят проверку прав пользователя — должны жить в приватной сети или service mesh.
</Warning>
